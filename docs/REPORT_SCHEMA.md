# Report Schema Reference

This document describes the complete annotated JSON schema for the unified audit report generated by Inspector.

---

## Top-Level Structure

```json
{
  "meta": { ... },
  "summary": { ... },
  "performance": { ... },
  "compliance": { ... },
  "ux": { ... },
  "telemetry": { ... },
  "workflowIntelligence": { ... },
  "sessionIntegrity": { ... },
  "apiHealth": { ... },
  "dataIntegrity": { ... },
  "offlineResilience": { ... },
  "longitudinalTrends": { ... }
}
```

---

## `meta` — Report Metadata

| Field | Type | Description | Example |
|---|---|---|---|
| `title` | string | Report title | `"Inspector — HIS/EMR Observability Report"` |
| `version` | string | Report schema version | `"4.0.0"` |
| `generatedAt` | string (ISO 8601) | When the report was generated | `"2026-06-19T10:30:00.000Z"` |
| `generatedAtFormatted` | string | Human-readable timestamp | `"6/19/2026, 10:30:00 AM"` |
| `sessionStartTime` | string (ISO 8601) | When the audit session started | `"2026-06-19T10:00:00.000Z"` |
| `sessionDuration` | string | Human-readable session duration | `"30m 0s"` |
| `sessionDurationMs` | number | Session duration in milliseconds | `1800000` |
| `electronVersion` | string | Electron runtime version | `"33.4.11"` |
| `targetURL` | string | Target EMR URL being audited | `"https://emr.example.com"` |
| `engine` | string | Audit engine description | `"Inspector Healthcare Browser (Electron/Chromium)"` |
| `totalMetrics` | number | Total performance metric entries | `156` |
| `totalComplianceChecks` | number | Total compliance check entries | `42` |
| `totalUXChecks` | number | Total UX event entries | `18` |
| `totalErrors` | number | Total error entries | `3` |

|---|---|---|---|

| `scores.performance.color` | string | Status color | `"amber"` |
| `overall` | number | Average of all scores (0-100) | `85` |
| `totalIssues` | number | Total deduplicated critical flags | `12` |
| `criticalFlags[]` | array | Array of critical flag objects | See below |

### criticalFlags[] Entry

| Field | Type | Description | Example |
|---|---|---|---|
| `severity` | string | Severity level | `"critical"`, `"warning"` |
| `type` | string | Flag type identifier | `"PHI_IN_STORAGE"` |
| `message` | string | Human-readable description | `"MRN found in localStorage"` |
| `timestamp` | string (ISO 8601) | When the flag was raised | `"2026-06-19T10:15:00.000Z"` |

**Flag types:** `PHI_IN_STORAGE`, `PHI_UNENCRYPTED`, `JCI_IPSG1_VIOLATION`, `AUTOLOGOFF_FAILURE`, `SIGNIFICANT_LAYOUT_SHIFT`, `CONSOLE_ERROR`, `EXPIRED_TOKEN_REQUEST`, `CONCURRENT_SESSION_ANOMALY`, `PRIVILEGE_SCOPE_EXCEEDED`, `REAUTHENTICATION_BYPASSED`, `SILENT_FAILURE`, `HIGH_ERROR_RATE`, `CLINICAL_API_ERROR`, `CLINICAL_SLA_BREACH`, `OFFLINE_WARNING_MISSING`, `RECONNECTION_SYNC_FAILURE`, `DEGRADED_MODE_SILENT_FREEZE`, `NO_OFFLINE_CACHE_FOR_CRITICAL_DATA`
